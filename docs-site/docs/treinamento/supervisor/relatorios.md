# Relatorios

Os relatorios sao ferramentas essenciais para analisar o desempenho da sua equipe, identificar tendencias e tomar decisoes baseadas em dados. Este guia ensina como gerar e interpretar os relatorios disponiveis.

## Acessando os Relatorios

1. Faca login no ChatBlue
2. Clique em **Relatorios** no menu lateral
3. Escolha o tipo de relatorio desejado

## Tipos de Relatorios

### 1. Relatorio de Desempenho da Equipe

Visao geral de como a equipe esta performando.

**O que mostra:**
- Total de tickets atendidos
- Taxa de SLA cumprido
- Tempo medio de resposta
- Avaliacao media
- Distribuicao por status

**Quando usar:**
- Reunioes de equipe
- Apresentacoes para gestao
- Analise semanal/mensal

### 2. Relatorio de SLA e Metricas

Foco no cumprimento dos acordos de nivel de servico.

**O que mostra:**
- % de SLA cumprido por tipo
- Violacoes detalhadas
- Tempos medios (resposta e resolucao)
- Tickets em risco

**Quando usar:**
- Identificar problemas de prazo
- Justificar necessidade de recursos
- Acompanhar metas

### 3. Relatorio Individual por Atendente

Desempenho de cada membro da equipe.

**O que mostra:**
- Metricas individuais
- Comparacao com media da equipe
- Evolucao ao longo do tempo
- Pontos fortes e de melhoria

**Quando usar:**
- Feedbacks individuais
- Avaliacoes de desempenho
- Identificar necessidades de treinamento

### 4. Relatorio de Tickets Detalhado

Lista completa de tickets com filtros avancados.

**O que mostra:**
- Todos os tickets do periodo
- Filtros por status, atendente, prioridade
- Detalhes de cada ticket
- Historico de acoes

**Quando usar:**
- Auditoria de atendimentos
- Investigar casos especificos
- Analise detalhada

### 5. Relatorio de Satisfacao do Cliente

Avaliacoes e feedbacks dos clientes.

**O que mostra:**
- Nota media geral
- Distribuicao das notas
- Comentarios dos clientes
- Comparacao por atendente

**Quando usar:**
- Medir qualidade do atendimento
- Identificar problemas recorrentes
- Reconhecer bom trabalho

### 6. Relatorio Comparativo de Periodos

Compare diferentes periodos de tempo.

**O que mostra:**
- Metricas lado a lado
- Variacao percentual
- Graficos de evolucao
- Tendencias

**Quando usar:**
- Avaliar progresso
- Identificar sazonalidade
- Medir impacto de mudancas

## Gerando um Relatorio

### Passo a Passo

1. **Selecione o tipo de relatorio**
   - Clique no card do relatorio desejado

2. **Configure os filtros**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Gerar Relatorio: Desempenho da Equipe                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  Periodo                                                                    │
│  ─────────────────────────────────────────────────────────────────────────  │
│  ( ) Hoje                                                                   │
│  ( ) Ontem                                                                  │
│  ( ) Ultimos 7 dias                                                         │
│  (●) Ultimos 30 dias                                                        │
│  ( ) Este mes                                                               │
│  ( ) Mes passado                                                            │
│  ( ) Personalizado: [__/__/____] ate [__/__/____]                          │
│                                                                              │
│  Filtros adicionais                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  Departamento: [Todos                    ▼]                                │
│  Atendente:    [Todos                    ▼]                                │
│  Prioridade:   [Todas                    ▼]                                │
│  Status:       [Todos                    ▼]                                │
│                                                                              │
│  Opcoes                                                                     │
│  ─────────────────────────────────────────────────────────────────────────  │
│  [x] Incluir graficos                                                       │
│  [x] Incluir tabelas detalhadas                                            │
│  [ ] Incluir lista de tickets                                              │
│  [x] Mostrar comparacao com periodo anterior                               │
│                                                                              │
│                            [Cancelar]  [Gerar Relatorio]                   │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

3. **Clique em "Gerar Relatorio"**
   - O relatorio sera processado e exibido

4. **Analise os dados**
   - Navegue pelas secoes do relatorio

5. **Exporte se necessario**
   - Clique em "Exportar" para salvar

## Analisando Tendencias

### Identificando Padroes

Ao analisar relatorios, procure por:

| Padrao | O Que Significa | Acao |
|--------|-----------------|------|
| **Volume aumentando** | Mais clientes ou mais problemas | Avaliar capacidade |
| **SLA caindo** | Equipe sobrecarregada ou lenta | Investigar causas |
| **Avaliacao caindo** | Qualidade diminuindo | Treinar equipe |
| **Picos em horarios** | Concentracao de demanda | Ajustar escalas |

### Perguntas para Guiar sua Analise

1. **Estamos melhorando ou piorando?**
   - Compare com periodos anteriores

2. **Onde estao os gargalos?**
   - Qual atendente, horario ou departamento tem mais problemas?

3. **O que causou mudancas?**
   - Houve algo diferente (promocao, problema tecnico)?

4. **O que podemos fazer diferente?**
   - Treinar? Contratar? Mudar processo?

### Exemplo Pratico

**Situacao**: SLA caiu de 95% para 90% no ultimo mes.

**Analise**:
1. Gerar relatorio de SLA
2. Identificar: 70% das violacoes sao no horario 10-12h
3. Verificar: volume de tickets nesse horario aumentou 30%

**Plano**:
- Realocar um atendente do turno da tarde para o horario de pico
- Avaliar uso de IA para triagem

**Acao**:
- Implementar mudanca na proxima semana
- Acompanhar metricas diariamente

**Resultado esperado**: SLA retornar a 95% em 2 semanas

## Dicas para Relatorios Eficazes

:::tip Regularidade
Gere relatorios com frequencia regular (diario, semanal, mensal) para acompanhar evolucao.
:::

:::tip Foco no Importante
Nao se perca em muitos numeros. Foque nas metricas principais: SLA, tempo de resposta, avaliacao.
:::

:::tip Acao, nao Apenas Dados
Relatorios devem levar a acoes. Se nao vai mudar nada, nao adianta so olhar os numeros.
:::

:::tip Compartilhe com a Equipe
Mostre os resultados para sua equipe. Transparencia motiva e engaja.
:::

## Perguntas Frequentes

### Posso ver dados de antes de eu ser supervisor?
Sim, voce tem acesso ao historico completo da empresa.

### Os atendentes veem esses relatorios?
Por padrao, nao. Mas voce pode compartilhar partes relevantes com eles.

### Os dados sao em tempo real?
Depende do relatorio. O dashboard e em tempo real. Relatorios exportados mostram dados ate o momento da geracao.

### Posso criar relatorios personalizados?
O sistema oferece filtros flexiveis. Para relatorios muito especificos, exporte para Excel e manipule os dados.
