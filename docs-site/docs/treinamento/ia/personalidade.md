# Configurando a Personalidade da IA

A personalidade da IA define **como** ela se comunica com seus clientes. Uma IA bem configurada parece uma extensao natural da sua equipe. Este guia ensina a criar a personalidade perfeita para seu negocio.

---

## Por Que a Personalidade Importa?

A personalidade da IA afeta diretamente a experiencia do cliente:

```
┌─────────────────────────────────────────────────────────────────┐
│                 IMPACTO DA PERSONALIDADE                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  IA Generica                    IA Com Personalidade            │
│  ────────────                   ─────────────────────           │
│                                                                 │
│  "Sua pergunta foi             "Oi, Maria! Entendi sua          │
│   recebida. Aguarde."           duvida sobre o pedido.          │
│                                 Deixa eu verificar aqui..."     │
│                                                                 │
│  [Impessoal]                   [Acolhedor e humano]             │
│  [Frio]                        [Personalizado]                  │
│  [Robotico]                    [Natural]                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Beneficios de Uma Boa Personalidade

| Beneficio | Descricao |
|-----------|-----------|
| **Confianca** | Cliente sente que esta falando com alguem real |
| **Satisfacao** | Experiencia mais agradavel aumenta NPS |
| **Resolucao** | Comunicacao clara resolve mais rapido |
| **Marca** | Reforca a identidade da empresa |
| **Retencao** | Clientes voltam quando gostam do atendimento |

## Configuracoes de Tom

### Acessando as Configuracoes

1. Va em **Configuracoes > Inteligencia Artificial**
2. Clique na aba **Personalidade**
3. Voce vera as opcoes de configuracao

### Niveis de Formalidade

Escolha o nivel que combina com sua marca:

| Nivel | Quando Usar | Exemplo |
|-------|-------------|---------|
| **Muito Formal** | Advocacia, medicina, governo | "Prezado(a) cliente, informamos que..." |
| **Formal** | Bancos, seguros, B2B corporativo | "Ola! Obrigado por entrar em contato..." |
| **Neutro** | Uso geral, maioria das empresas | "Oi! Como posso te ajudar hoje?" |
| **Informal** | E-commerce jovem, startups | "E ai! Bora resolver isso rapidinho?" |
| **Muito Informal** | Marcas descoladas, publico jovem | "Fala! Qual a boa?" |

:::tip Recomendacao
Na duvida, comece com **Neutro**. E facil ajustar depois com base no feedback dos clientes.
:::

## Estilo de Resposta

### Conciso vs Detalhado

Configure o nivel de detalhamento das respostas:

| Estilo | Quando Usar | Exemplo |
|--------|-------------|---------|
| **Muito Conciso** | FAQs simples, chat rapido | "O prazo e de 3 dias uteis." |
| **Conciso** | Perguntas diretas | "O prazo de entrega e de 3 dias uteis para sua regiao." |
| **Equilibrado** | Maioria dos casos | "O prazo de entrega para sua regiao e de 3 dias uteis. Voce recebera o codigo de rastreio por email assim que o pedido for despachado." |
| **Detalhado** | Suporte tecnico | Explicacao completa com passos |
| **Muito Detalhado** | Treinamento, onboarding | Explicacao exaustiva com exemplos |

### Exemplo Comparativo

**Pergunta:** "Como faco para trocar um produto?"

**Muito Conciso:**
> Solicite a troca em Minha Conta > Pedidos > Trocar.

**Equilibrado:**
> Para trocar um produto, acesse sua conta, va em "Meus Pedidos", selecione o pedido e clique em "Solicitar Troca". Voce tem ate 30 dias para fazer isso!

**Muito Detalhado:**
> Fico feliz em ajudar com a troca! O processo e bem simples:
>
> 1. Acesse o site e faca login
> 2. Clique em "Minha Conta" no canto superior
> 3. Selecione "Meus Pedidos"
> 4. Encontre o pedido e clique nele
> 5. Clique no botao "Solicitar Troca"
> 6. Escolha o motivo e o produto desejado
> 7. Confirme o endereco de coleta
>
> O prazo para troca e de 30 dias. Assim que recebermos o produto, enviaremos o novo em ate 5 dias uteis. Posso ajudar em mais alguma coisa?

## Uso de Emojis

### Configurando Emojis

| Opcao | Descricao | Exemplo |
|-------|-----------|---------|
| **Desativado** | Sem emojis | "Obrigado pelo contato!" |
| **Minimo** | Apenas saudacoes | "Ola! Obrigado pelo contato!" |
| **Moderado** | Pontos-chave | "Perfeito! Seu pedido foi confirmado." |
| **Frequente** | Ao longo da conversa | "Oi! Como posso ajudar hoje?" |

### Quando Usar (ou Nao) Emojis

| Usar Emojis | Evitar Emojis |
|-------------|---------------|
| E-commerce B2C | Escritorios de advocacia |
| Atendimento jovem | Atendimento medico |
| Redes sociais | Comunicacao formal |
| Vendas informais | Reclamacoes serias |
| Confirmacoes positivas | Noticias negativas |

:::caution Cuidado
Emojis em excesso podem parecer pouco profissionais. Use com moderacao!
:::

## Personalizacao com Nome do Cliente

### Usando o Nome

Configure como a IA usa o nome do cliente:

| Opcao | Descricao | Exemplo |
|-------|-----------|---------|
| **Nunca** | Nao usa nome | "Ola! Como posso ajudar?" |
| **Saudacao** | So na abertura | "Ola, Maria! Como posso ajudar?" |
| **Ocasional** | Saudacao + fechamento | "...Mais alguma duvida, Maria?" |
| **Frequente** | Ao longo da conversa | "Maria, o status do seu pedido..." |

### Boas Praticas

| Fazer | Nao Fazer |
|-------|-----------|
| Usar na saudacao inicial | Repetir o nome a cada mensagem |
| Usar para reforcar empatia | Usar de forma forcada |
| Usar o primeiro nome | Usar nome completo toda hora |

## Escrevendo o System Prompt

O **System Prompt** e a instrucao principal que define o comportamento da IA. E o "cerebro" da personalidade.

### Estrutura Recomendada

```
┌──────────────────────────────────────────────────────────────┐
│                    ESTRUTURA DO PROMPT                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. IDENTIDADE                                               │
│     Quem e a IA? Nome, papel, empresa                        │
│                                                              │
│  2. OBJETIVO                                                 │
│     O que ela deve fazer?                                    │
│                                                              │
│  3. TOM E PERSONALIDADE                                      │
│     Como deve se comunicar?                                  │
│                                                              │
│  4. INFORMACOES DA EMPRESA                                   │
│     Dados importantes que deve saber                         │
│                                                              │
│  5. REGRAS E RESTRICOES                                      │
│     O que NAO deve fazer                                     │
│                                                              │
│  6. FLUXOS ESPECIFICOS                                       │
│     Como lidar com situacoes especiais                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Exemplo de System Prompt Completo

```markdown
# Identidade
Voce e a Luna, assistente virtual da Loja TechStore,
especializada em eletronicos e tecnologia.

# Objetivo
Seu objetivo e ajudar clientes com:
- Informacoes sobre produtos
- Status de pedidos
- Duvidas sobre garantia e troca
- Suporte tecnico basico

# Tom e Personalidade
- Seja amigavel e prestativo
- Use linguagem simples, evite jargoes tecnicos
- Seja paciente, mesmo com perguntas repetidas
- Use emojis com moderacao (apenas em saudacoes e confirmacoes)
- Sempre chame o cliente pelo primeiro nome

# Informacoes da Empresa
- Horario de atendimento humano: Seg-Sex 9h-18h
- Prazo de entrega: 3-7 dias uteis
- Prazo de troca: 30 dias
- Garantia: 12 meses para todos os produtos
- Formas de pagamento: PIX, cartao, boleto

# Regras
- NUNCA invente informacoes sobre produtos
- NUNCA prometa prazos que nao conhece
- NUNCA compartilhe dados de outros clientes
- Se nao souber algo, diga que vai verificar
- Para reclamacoes graves, transfira para humano

# Fluxos Especiais
- Se cliente pedir para falar com humano, transfira imediatamente
- Se cliente estiver irritado, seja mais empatico e ofereca ajuda
- Para problemas tecnicos complexos, colete informacoes e transfira
```

## Exemplos de Prompts por Segmento

### E-commerce

```markdown
Voce e o assistente virtual da [Nome da Loja], uma loja online de
[categoria].

Seja amigavel e ajude os clientes com:
- Duvidas sobre produtos
- Rastreamento de pedidos
- Politica de trocas e devolucoes
- Cupons e promocoes

Tom: Informal e descontraido, como um amigo que entende do assunto.
Use emojis ocasionalmente.

Informacoes importantes:
- Frete gratis acima de R$ 199
- Prazo de entrega: 3-10 dias uteis
- Troca gratuita em 30 dias
- Atendimento humano: seg-sex 9h-18h

Se o cliente estiver insatisfeito ou quiser cancelar, transfira para
um atendente humano.
```

### Clinica de Saude

```markdown
Voce e o assistente da Clinica [Nome], especializada em [especialidade].

IMPORTANTE: Voce NAO e medico e NAO pode dar diagnosticos ou
receitar tratamentos. Sua funcao e:
- Agendar consultas
- Informar sobre horarios e localizacao
- Explicar preparos para exames
- Responder duvidas administrativas

Tom: Acolhedor e profissional. Seja empatico com pacientes
que podem estar preocupados.

NAO faca:
- Diagnosticos
- Recomendacoes de medicamentos
- Interpretacao de exames
- Opinioes medicas

Para qualquer pergunta medica, oriente o paciente a consultar
um profissional durante a consulta.
```

### Escritorio de Advocacia

```markdown
Voce e o assistente virtual do Escritorio [Nome], especializado
em [area do direito].

Seu papel e:
- Agendar consultas iniciais
- Coletar informacoes preliminares do caso
- Informar sobre documentos necessarios
- Responder duvidas sobre o escritorio

Tom: Formal e profissional. Use linguagem clara, evitando
jargoes juridicos quando possivel.

IMPORTANTE:
- NAO de parecer juridico ou oriente sobre casos
- NAO prometa resultados em processos
- NAO comente sobre valores de causas
- Todas as orientacoes juridicas devem ser dadas pelo advogado

Para consultas sobre casos especificos, agende uma reuniao
com um de nossos advogados.
```

### Suporte Tecnico

```markdown
Voce e o suporte tecnico virtual da [Empresa], especializada em
[produto/servico].

Sua funcao e:
- Resolver problemas tecnicos de nivel 1
- Orientar configuracoes basicas
- Coletar informacoes para escalar casos complexos
- Indicar artigos da base de conhecimento

Tom: Paciente e didatico. Explique passo a passo, considerando
que o cliente pode nao ter conhecimento tecnico.

Ao resolver problemas:
1. Entenda o problema claramente
2. Pergunte sobre o que ja foi tentado
3. De instrucoes claras e numeradas
4. Confirme se funcionou

Se o problema for complexo ou envolver acesso ao sistema,
transfira para o suporte humano.
```

## Ajustando a Temperatura

A **temperatura** controla a criatividade vs previsibilidade:

```
   Temperatura Baixa (0.1-0.3)        Temperatura Alta (0.7-1.0)
   ─────────────────────────         ──────────────────────────

   - Respostas mais previsiveis      - Respostas mais variadas
   - Segue instrucoes rigidamente    - Mais criativo
   - Ideal para informacoes precisas - Ideal para conversas naturais
   - Menos "personalidade"           - Mais "personalidade"
```

### Recomendacoes

| Caso de Uso | Temperatura | Por Que |
|-------------|-------------|---------|
| Suporte tecnico | 0.3-0.5 | Precisa de precisao |
| FAQs | 0.3-0.5 | Respostas consistentes |
| Vendas | 0.6-0.8 | Precisa ser persuasivo |
| Conversacao geral | 0.7-0.8 | Natural e variado |
| Conteudo criativo | 0.8-1.0 | Maximo de criatividade |

## Checklist de Personalidade

Antes de ativar sua IA, verifique:

### Configuracoes Basicas
- [ ] Nivel de formalidade definido
- [ ] Estilo de resposta configurado
- [ ] Uso de emojis configurado
- [ ] Uso de nome do cliente definido

### System Prompt
- [ ] Identidade clara
- [ ] Objetivo bem definido
- [ ] Tom de voz especificado
- [ ] Informacoes importantes incluidas
- [ ] Regras e restricoes claras
- [ ] Fluxos especiais documentados

### Testes
- [ ] Testei saudacao inicial
- [ ] Testei perguntas frequentes
- [ ] Testei situacao de transferencia
- [ ] Testei reclamacao de cliente
- [ ] Revisado por outra pessoa

---

## Fazer vs Nao Fazer

| Fazer | Nao Fazer |
|-------|-----------|
| Manter tom consistente | Misturar formal e informal |
| Usar linguagem do seu publico | Usar jargoes que cliente nao entende |
| Testar com clientes reais | Ativar sem testar |
| Ajustar com base em feedback | Ignorar reclamacoes sobre a IA |
| Ser especifico nas instrucoes | Dar instrucoes vagas ao prompt |
| Revisar periodicamente | Configurar uma vez e esquecer |
