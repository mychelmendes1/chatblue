# Seguranca e Privacidade

Proteger dados dos clientes nao e apenas uma obrigacao legal - e uma questao de confianca. Este guia ensina como manter a seguranca no seu dia a dia de atendimento.

## Protecao de Dados

### Dados que NUNCA Devem Ser Compartilhados

| Tipo de Dado | Exemplos | Risco |
|--------------|----------|-------|
| **Senhas** | Senha de conta, PIN | Invasao de conta |
| **Cartao completo** | Numero + CVV + validade | Fraude financeira |
| **Dados bancarios** | Conta, agencia, senhas | Roubo de dinheiro |
| **Documentos** | CPF, RG (completos) | Fraude de identidade |
| **Dados de outros clientes** | Qualquer informacao | Violacao de privacidade |

### O Que Voce Pode Fazer

| Situacao | Certo | Errado |
|----------|-------|--------|
| Cliente pede CPF de outra pessoa | "Nao posso informar dados de terceiros" | Dar o CPF |
| Cliente quer confirmar cartao | "Por seguranca, nao temos acesso ao numero completo" | Ler o numero |
| Cliente esqueceu senha | "Vou enviar um link de redefinicao por email" | Dar uma senha temporaria por chat |

---

## O Que NAO Fazer - Lista Critica

### NUNCA Faca Isso

| Acao Proibida | Por Que |
|---------------|---------|
| Anotar senhas de clientes | Risco de vazamento |
| Compartilhar sua senha | Outro pode agir como voce |
| Acessar conta sem autorizacao | Violacao de privacidade |
| Copiar dados para dispositivo pessoal | Risco de vazamento |
| Compartilhar tela com dados sensiveis | Exposicao desnecessaria |
| Tirar print de dados de cliente | Pode vazar |
| Enviar dados por canais nao oficiais | Sem rastreabilidade |
| Comentar casos com terceiros | Violacao de sigilo |

### Sinais de Alerta

Desconfie quando:

- Cliente pede informacoes de outras contas
- Alguem se passa por "supervisor" pedindo acesso
- Email/telefone nao bate com o cadastro
- Pedidos urgentes para ignorar procedimentos
- Solicitacao de mudancas criticas sem verificacao

## Seguranca da Sua Conta

### Boas Praticas de Senha

| Fazer | Nao Fazer |
|-------|-----------|
| Senha com 12+ caracteres | Senhas curtas (123456) |
| Misturar letras, numeros, simbolos | Apenas letras |
| Senha unica para o trabalho | Reusar senha pessoal |
| Trocar periodicamente | Manter a mesma por anos |
| Usar gerenciador de senhas | Anotar em papel |

### Exemplo de Senha Forte

```
❌ Senhas Fracas:
   - 123456
   - senha123
   - joao1990
   - empresa@1

✅ Senhas Fortes:
   - Tr@b4lh0-S3gur0!2024
   - Ch@tBlue_Atend1m3nto#
   - M!nh@S3nh@-Trab@lh0
```

### Protegendo seu Acesso

| Pratica | Como Fazer |
|---------|-----------|
| **Bloqueio de tela** | Sempre que sair da mesa |
| **Logout** | Ao terminar o expediente |
| **2FA** | Ativar autenticacao em dois fatores |
| **Nao salvar senha** | Em computadores compartilhados |
| **Cuidado com Wi-Fi** | Evitar redes publicas |

## Reconhecendo Phishing

### O Que e Phishing?

Phishing e uma tentativa de golpe onde criminosos tentam obter informacoes sensiveis fingindo ser alguem de confianca.

### Sinais de Phishing

```
┌──────────────────────────────────────────────────────────────┐
│                 IDENTIFICANDO PHISHING                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  EMAILS SUSPEITOS                                            │
│  ────────────────                                            │
│  [!] Remetente estranho (suporte@empresa-com.xyz)           │
│  [!] Erros de portugues                                      │
│  [!] Urgencia exagerada ("AGORA ou perdera acesso")         │
│  [!] Links diferentes do texto                              │
│  [!] Anexos inesperados                                      │
│  [!] Pede senha ou dados sensiveis                          │
│                                                              │
│  MENSAGENS SUSPEITAS NO CHAT                                │
│  ──────────────────────────                                 │
│  [!] "Sou do suporte, preciso da sua senha"                 │
│  [!] "Urgente: clique neste link agora"                     │
│  [!] Links encurtados suspeitos                             │
│  [!] Promessas boas demais (premio, dinheiro)               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Exemplos de Phishing

**Email falso:**
```
De: suporte@chatblue-verificacao.com  ← Dominio errado!
Assunto: URGENTE: Sua conta sera bloqueada!

Prezado usuario,
Detectamos atividade suspeita. Clique aqui para
confirmar seus dados ou sua conta sera bloqueada
em 24 horas!

[CLIQUE AQUI PARA VERIFICAR]  ← Link suspeito!
```

**Cliente suspeito:**
```
"Ola, sou do departamento de TI. Estamos fazendo
manutencao e preciso que voce me passe sua senha
para verificar seu acesso. E urgente!"
```

### O Que Fazer

1. **NAO clique** em links suspeitos
2. **NAO informe** senhas ou dados
3. **Verifique** com seu supervisor
4. **Reporte** ao departamento de seguranca
5. **Delete** emails suspeitos

## Privacidade e LGPD

### O Que e LGPD?

A **Lei Geral de Protecao de Dados** (LGPD) regulamenta como empresas devem tratar dados pessoais no Brasil.

### Seus Deveres pela LGPD

| Dever | O Que Significa |
|-------|-----------------|
| **Minimizacao** | Coletar apenas dados necessarios |
| **Finalidade** | Usar dados apenas para o proposito original |
| **Transparencia** | Informar como dados serao usados |
| **Seguranca** | Proteger dados contra vazamentos |
| **Retencao** | Nao manter dados alem do necessario |

### Direitos do Cliente (que voce deve respeitar)

- Saber quais dados temos
- Corrigir dados incorretos
- Solicitar exclusao de dados
- Revogar consentimento
- Portabilidade de dados

### Na Pratica

| Situacao | Como Agir |
|----------|-----------|
| Cliente pede para ver seus dados | Direcionar ao canal oficial |
| Cliente quer excluir conta | Seguir procedimento de exclusao |
| Cliente pergunta por que pedimos X | Explicar finalidade |
| Voce precisa de um dado | Pedir apenas o necessario |

## Procedimentos de Seguranca

### Verificacao de Identidade

Antes de dar informacoes sensiveis:

```
┌──────────────────────────────────────────────────────────────┐
│              PROCEDIMENTO DE VERIFICACAO                     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. CONFIRMAR IDENTIDADE                                     │
│     - Nome completo                                          │
│     - CPF (ultimos 4 digitos)                               │
│     - Data de nascimento                                     │
│     - Email cadastrado                                       │
│                                                              │
│  2. VERIFICAR NO SISTEMA                                     │
│     - Dados batem com cadastro?                             │
│     - Numero de contato e o mesmo?                          │
│                                                              │
│  3. EM CASO DE DUVIDA                                        │
│     - Fazer pergunta de seguranca                           │
│     - Pedir para acessar por canal verificado               │
│     - Consultar supervisor                                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Quando Escalar

Chame seu supervisor em:

- Suspeita de fraude
- Tentativa de phishing
- Cliente pedindo acesso indevido
- Vazamento de dados
- Qualquer situacao que pareca errada

## Seguranca no Dia a Dia

### Checklist Diario

**Inicio do Dia:**
- [ ] Verificar se ninguem usou minha conta
- [ ] Checar se 2FA esta ativo
- [ ] Confirmar que estou em rede segura

**Durante o Dia:**
- [ ] Bloquear tela ao sair da mesa
- [ ] Nao compartilhar tela com dados sensiveis
- [ ] Verificar identidade antes de dar informacoes

**Final do Dia:**
- [ ] Fazer logout do sistema
- [ ] Nao deixar anotacoes com dados visiveis
- [ ] Verificar se nao salvei dados em local errado

### Ambiente Seguro

| Fazer | Nao Fazer |
|-------|-----------|
| Trabalhar em local privado | Atender em locais publicos |
| Usar fones para audios | Reproduzir audios em voz alta |
| Bloquear tela ao sair | Deixar sistema aberto |
| Usar rede da empresa | Usar Wi-Fi publico |
| Descartar papeis corretamente | Jogar no lixo comum |

## Incidentes de Seguranca

### O Que e um Incidente?

- Vazamento de dados
- Acesso nao autorizado
- Perda de dispositivo
- Compartilhamento indevido
- Ataque de phishing bem-sucedido

### Como Reportar

```
┌──────────────────────────────────────────────────────────────┐
│            FLUXO DE REPORTE DE INCIDENTE                     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. IDENTIFICAR                                              │
│     O que aconteceu? Quando? Quem foi afetado?              │
│                                                              │
│  2. CONTER                                                   │
│     Pare o que estava fazendo. Nao tente "consertar".       │
│                                                              │
│  3. REPORTAR IMEDIATAMENTE                                   │
│     Avise seu supervisor e/ou seguranca da informacao.      │
│                                                              │
│  4. DOCUMENTAR                                               │
│     Anote tudo que lembrar. Isso ajuda na investigacao.     │
│                                                              │
│  5. COLABORAR                                                │
│     Ajude na investigacao. Seja transparente.               │
│                                                              │
│  IMPORTANTE: Nao tente esconder! Quanto antes soubermos,    │
│  menor o dano.                                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## Compliance - Conformidade

### Regras Gerais

| Regra | Descricao |
|-------|-----------|
| Confidencialidade | Dados de clientes sao confidenciais |
| Uso autorizado | So acesse sistemas permitidos |
| Registro | Acoes sao registradas para auditoria |
| Treinamento | Participe de treinamentos de seguranca |
| Atualizacao | Mantenha-se informado sobre novas regras |

### Consequencias de Violacoes

| Gravidade | Exemplo | Consequencia |
|-----------|---------|--------------|
| Leve | Deixar tela desbloqueada | Advertencia |
| Media | Compartilhar dados sem verificacao | Suspensao |
| Grave | Vazar dados intencionalmente | Demissao + processo |

## Fazer vs Nao Fazer

| Fazer | Nao Fazer |
|-------|-----------|
| Verificar identidade | Confiar cegamente |
| Usar senhas fortes | Reutilizar senhas |
| Bloquear tela | Deixar aberto |
| Reportar incidentes | Esconder problemas |
| Questionar pedidos suspeitos | Obedecer sem pensar |
| Proteger dados do cliente | Compartilhar livremente |
| Seguir procedimentos | Criar atalhos arriscados |
| Perguntar quando em duvida | Assumir que esta certo |

## Quiz de Seguranca

Teste seus conhecimentos:

**1. Um cliente pede a senha dele que esqueceu. Voce:**
- a) Diz a senha que esta no sistema
- b) Cria uma nova senha e passa por chat
- c) Envia link de redefinicao por email cadastrado ✓

**2. Recebe email pedindo para confirmar sua senha. Voce:**
- a) Responde com a senha
- b) Clica no link e preenche
- c) Ignora e reporta ao TI ✓

**3. Colega pede sua senha "so para ver uma coisa". Voce:**
- a) Passa, e colega de confianca
- b) Deixa ele usar seu computador logado
- c) Recusa e orienta a usar proprio acesso ✓

**4. Cliente pede dados de outro cliente. Voce:**
- a) Verifica se sao parentes e passa
- b) Passa se parecer urgente
- c) Recusa e explica politica de privacidade ✓
