# Eficiencia no Atendimento

Ser eficiente significa fazer mais com menos - atender mais clientes, em menos tempo, com a mesma qualidade. Este guia ensina tecnicas praticas para aumentar sua produtividade no ChatBlue.

## Gestao do Tempo

### Priorizacao de Tickets

Nem todos os tickets sao iguais. Aprenda a priorizar:

| Prioridade | Criterio | Acao |
|------------|----------|------|
| **URGENTE** | SLA proximo de estourar | Atender imediatamente |
| **ALTA** | Cliente VIP ou reclamacao | Atender em seguida |
| **MEDIA** | Pergunta normal | Ordem da fila |
| **BAIXA** | Informacao simples | Pode esperar se necessario |

### Tecnica Pomodoro Adaptada

Divida seu tempo em blocos focados:

| Bloco | Duracao | Atividade |
|-------|---------|-----------|
| **Foco** | 25 min | Atender tickets sem distracao |
| **Pausa** | 5 min | Descanso rapido |
| **Foco** | 25 min | Mais atendimentos |
| **Pausa Longa** | 15 min | A cada 4 blocos |

## Respostas Rapidas

### O Que Sao?

Respostas rapidas sao **mensagens pre-configuradas** que voce envia com poucos cliques. Economizam tempo em perguntas frequentes.

### Acessando Respostas Rapidas

1. No campo de mensagem, digite `/`
2. Ou clique no icone de **raio**
3. Selecione a resposta desejada
4. Edite se necessario
5. Envie

```
┌──────────────────────────────────────────────────────────────┐
│  Respostas Rapidas                                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Buscar: [________________]                                  │
│                                                              │
│  /saudacao                                                   │
│  "Ola, {{nome}}! Como posso ajudar?"                        │
│                                                              │
│  /prazo                                                      │
│  "O prazo de entrega e de 3 a 7 dias uteis..."              │
│                                                              │
│  /rastreio                                                   │
│  "Para rastrear seu pedido, use o codigo..."                │
│                                                              │
│  /troca                                                      │
│  "Para solicitar uma troca, acesse..."                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Criando Respostas Rapidas Eficazes

| Caracteristica | Descricao |
|----------------|-----------|
| **Curtas e objetivas** | Vai direto ao ponto |
| **Personalizaveis** | Usa variaveis como {{nome}} |
| **Atualizadas** | Informacoes sempre corretas |
| **Organizadas** | Nomeadas de forma logica |

### Respostas Rapidas Essenciais

| Atalho | Uso | Modelo |
|--------|-----|--------|
| `/ola` | Saudacao inicial | "Ola, {{nome}}! Tudo bem? Como posso ajudar?" |
| `/prazo` | Prazo de entrega | "O prazo de entrega e de 3 a 7 dias uteis..." |
| `/rastrear` | Como rastrear | "Para rastrear, use o codigo no site dos Correios..." |
| `/troca` | Politica de troca | "Voce pode trocar em ate 30 dias..." |
| `/pix` | Dados para PIX | "Para pagar via PIX, use a chave..." |
| `/horario` | Horario de funcionamento | "Atendemos de segunda a sexta, 9h as 18h..." |
| `/aguarde` | Pedir espera | "Um momento, por favor. Estou verificando..." |
| `/obrigado` | Agradecimento final | "Foi um prazer ajudar! Qualquer duvida, volte!" |

## Multitarefa Inteligente

### Atendendo Multiplos Clientes

Voce pode atender varios clientes ao mesmo tempo de forma eficiente:

```
┌──────────────────────────────────────────────────────────────┐
│                 FLUXO DE MULTITAREFA                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   Cliente A                Cliente B              Cliente C  │
│   pergunta                 pergunta               aguarda    │
│      │                        │                      │       │
│      ▼                        │                      │       │
│   Responde A                  │                      │       │
│   (resposta rapida)           │                      │       │
│      │                        ▼                      │       │
│      │                    Responde B                 │       │
│      │                    (pesquisa info)            │       │
│      │                        │                      ▼       │
│      │                        │                  Atualiza C  │
│      ▼                        │                  (status)    │
│   A responde                  ▼                      │       │
│      │                    B responde                 │       │
│      ▼                        │                      │       │
│   Continua A                  ▼                      │       │
│                           Continua B                 │       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Dicas para Multitarefa

| Fazer | Nao Fazer |
|-------|-----------|
| Responder mensagem simples primeiro | Deixar cliente esperando muito |
| Usar tempo de espera de um para atender outro | Confundir conversas |
| Organizar por tabs/grupos | Abrir dezenas de conversas |
| Avisar quando precisar de tempo | Sumir sem explicacao |
| Anotar onde parou | Confiar na memoria |

### Quando NAO Fazer Multitarefa

- Cliente muito irritado (precisa de foco total)
- Problema complexo (risco de erro)
- Quando estiver cansado
- Situacoes delicadas

## Atalhos de Teclado

Aprenda os atalhos para ser mais rapido:

### Atalhos Essenciais

| Atalho | Acao |
|--------|------|
| `Enter` | Enviar mensagem |
| `Shift + Enter` | Nova linha |
| `Ctrl + K` | Buscar conversas |
| `/` | Abrir respostas rapidas |
| `Ctrl + /` | Ver todos os atalhos |
| `Esc` | Fechar paineis |

### Atalhos de Navegacao

| Atalho | Acao |
|--------|------|
| `Alt + ↑` | Conversa anterior |
| `Alt + ↓` | Proxima conversa |
| `Ctrl + 1-9` | Ir para aba especifica |

### Atalhos de Acao

| Atalho | Acao |
|--------|------|
| `Ctrl + T` | Transferir ticket |
| `Ctrl + R` | Resolver ticket |
| `Ctrl + N` | Nova nota interna |
| `Ctrl + E` | Editar contato |

:::tip Memorize Gradualmente
Nao tente decorar todos de uma vez! Comece com 3-5 atalhos que mais usa e va adicionando aos poucos.
:::

## Evitando Erros Comuns

### Erros que Desperdicam Tempo

| Erro | Consequencia | Solucao |
|------|--------------|---------|
| Responder sem ler tudo | Precisa perguntar de novo | Ler mensagem completa |
| Nao verificar historico | Repetir informacoes | Checar conversas anteriores |
| Enviar informacao errada | Retrabalho e confusao | Confirmar antes de enviar |
| Esquecer de registrar | Perder contexto | Usar notas internas |
| Nao usar respostas rapidas | Redigitar sempre | Criar e usar templates |

### Checklist Anti-Erro

Antes de responder:

- [ ] Li toda a mensagem do cliente?
- [ ] Verifiquei o historico?
- [ ] A informacao que vou dar esta correta?
- [ ] Revisei a mensagem?

Antes de transferir:

- [ ] Coletei todas as informacoes necessarias?
- [ ] Expliquei ao cliente o que vai acontecer?
- [ ] Adicionei nota para o proximo atendente?

## Organizacao Pessoal

### Inicio do Dia

1. Checar tickets pendentes da noite
2. Revisar SLAs proximos de estourar
3. Organizar prioridades do dia
4. Preparar respostas rapidas que vai precisar

### Durante o Dia

| Hora | Atividade |
|------|-----------|
| Primeira hora | Tickets mais urgentes |
| Meio do dia | Tickets regulares |
| Final do dia | Follow-ups e organizacao |

### Final do Dia

1. Resolver ou transferir tickets abertos
2. Adicionar notas em casos pendentes
3. Limpar organizacao da tela
4. Preparar briefing para o dia seguinte

## Trabalhando com a IA

### Deixe a IA Ajudar

| A IA Pode | Voce Foca Em |
|-----------|--------------|
| Responder perguntas simples | Casos complexos |
| Fazer triagem inicial | Resolver problemas |
| Coletar informacoes basicas | Decisoes importantes |
| Atender fora do horario | Atendimento personalizado |

### Assumindo da IA

Quando assumir um ticket que a IA atendeu:

1. **Leia o historico** - Veja o que a IA ja perguntou
2. **Nao repita** - Use as informacoes ja coletadas
3. **Continue de onde parou** - Nao recomece do zero
4. **Seja natural** - O cliente sabe que e humano agora

### O Que Acompanhar

| Metrica | O Que Mede | Meta Sugerida |

| **Tickets/hora** | Produtividade | 8-12 tickets |
