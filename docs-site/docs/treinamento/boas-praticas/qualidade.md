# Qualidade no Atendimento

Qualidade nao e apenas resolver o problema - e resolver da melhor forma possivel, deixando o cliente satisfeito e com vontade de voltar. Este guia ensina como elevar o padrao do seu atendimento.

## Resolucao no Primeiro Contato (FCR)

### O Que e FCR?

**First Contact Resolution** (Resolucao no Primeiro Contato) e quando o cliente tem seu problema resolvido na primeira interacao, sem precisar voltar a entrar em contato.

### Por Que FCR Importa?

| Alto FCR | Baixo FCR |
|----------|-----------|
| Cliente satisfeito | Cliente frustrado |
| Menos tickets | Mais tickets |
| Equipe eficiente | Retrabalho |
| Custos menores | Custos maiores |
| NPS alto | NPS baixo |

### Como Aumentar seu FCR

| Pratica | Como Fazer |
|---------|-----------|
| **Entender completamente** | Leia tudo, pergunte se necessario |
| **Pesquisar antes de responder** | Verifique a base de conhecimento |
| **Dar resposta completa** | Antecipe proximas duvidas |
| **Confirmar resolucao** | "Isso resolve sua duvida?" |
| **Deixar porta aberta** | "Qualquer coisa, volte!" |

### Exemplo: Antes e Depois

**Antes (FCR Baixo):**
```
Cliente: Qual o prazo de entrega?
Atendente: 3 a 7 dias.
Cliente: E como rastreio?
Atendente: No site dos Correios.
Cliente: Qual o codigo?
Atendente: Vou verificar...
```
*3 interacoes para resolver*

**Depois (FCR Alto):**
```
Cliente: Qual o prazo de entrega?
Atendente: O prazo e de 3 a 7 dias uteis! Seu codigo
de rastreio e AB123456789BR. Voce pode acompanhar no
site dos Correios (correios.com.br) ou pelo app.
O pedido ja foi despachado ontem!
```
*1 interacao resolve tudo*

## Informacao Completa

### Antecipando Necessidades

Bons atendentes respondem o que foi perguntado. **Otimos** atendentes respondem o que o cliente **vai** perguntar.

| Pergunta | Resposta Basica | Resposta Completa |
|----------|-----------------|-------------------|
| "Qual o prazo?" | "3 dias" | "3 dias uteis. Voce recebe o codigo de rastreio por email assim que despacharmos." |
| "Posso trocar?" | "Sim" | "Sim! Voce tem 30 dias. Acesse Minha Conta > Pedidos > Trocar. A primeira troca e gratis!" |
| "Tem desconto?" | "Temos promocoes" | "Temos 10% OFF na primeira compra! Use BEMVINDO10 no carrinho. Vale ate amanha!" |

## Documentacao do Atendimento

### Por Que Documentar?

- Outros atendentes sabem o que aconteceu
- Evita que cliente repita informacoes
- Historico para analise
- Protecao em caso de disputa

### O Que Documentar

| Tipo | Quando | Exemplo |
|------|--------|---------|
| **Notas internas** | Informacoes para equipe | "Cliente pediu retorno ate 17h" |
| **Resumo do problema** | Casos complexos | "Problema: produto chegou danificado" |
| **Acoes tomadas** | Ao resolver | "Solicitei reenvio - protocolo 12345" |
| **Pendencias** | Ao transferir | "Aguardando aprovacao do supervisor" |

### Como Escrever Boas Notas

**Ruim:**
> Falei com cliente

**Bom:**
> Cliente Maria solicitou troca do pedido #1234 por tamanho errado (P ao inves de M). Gerado protocolo de troca #5678. Coleta agendada para 20/01. Cliente ciente que novo produto chega em 5 dias uteis apos coleta.

### Template de Nota

```
RESUMO: [O que aconteceu em 1 frase]
PROBLEMA: [Qual era o problema do cliente]
SOLUCAO: [O que foi feito para resolver]
PROXIMOS PASSOS: [Se houver pendencia]
OBSERVACOES: [Detalhes importantes]
```

## Procedimentos de Handoff

**Handoff** e quando voce passa o atendimento para outra pessoa (transferencia, troca de turno, etc.).

### Handoff Bem Feito

```
┌──────────────────────────────────────────────────────────────┐
│                    PROCESSO DE HANDOFF                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  QUEM TRANSFERE                    QUEM RECEBE               │
│  ────────────────                  ─────────────             │
│                                                              │
│  1. Coletar todas                  1. Ler historico          │
│     as informacoes                    completo               │
│                                                              │
│  2. Registrar em nota              2. Ler nota interna       │
│     interna                                                  │
│                                                              │
│  3. Avisar o cliente               3. Continuar de onde      │
│     sobre a transferencia             parou                  │
│                                                              │
│  4. Garantir que o                 4. NAO pedir informacoes  │
│     proximo tem tudo                  que ja foram dadas     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### O Que Incluir na Transferencia

| Item | Por Que |
|------|---------|
| Resumo do problema | Para entender rapidamente |
| O que ja foi tentado | Para nao repetir |
| Dados do cliente | Nome, contato, pedido |
| Urgencia | Para priorizar |
| Expectativa do cliente | O que ele espera |

### Exemplo de Transferencia

**Mensagem para o cliente:**
> Maria, para resolver essa questao da melhor forma, vou te transferir para nossa equipe de Trocas que e especializada nisso. Eles ja tem todas as informacoes e vao continuar de onde paramos. Um momento!

**Nota interna:**
> TRANSFERENCIA: Trocas
> CLIENTE: Maria Silva - Pedido #1234
> PROBLEMA: Recebeu tamanho errado (P ao inves de M)
> JA TENTADO: Verificar estoque (tem M disponivel)
> PROXIMA ACAO: Gerar protocolo de troca
> URGENCIA: Media (cliente precisa do produto para evento dia 25)
> OBSERVACAO: Cliente bem compreensiva, nao esta irritada

## Satisfacao do Cliente (CSAT)

### Medindo Satisfacao

O CSAT (Customer Satisfaction Score) e a nota que o cliente da ao atendimento.

```
┌──────────────────────────────────────────────────────────────┐
│                    ESCALA DE CSAT                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   1        2        3        4        5                      │
│   ●        ●        ●        ●        ●                      │
│   │        │        │        │        │                      │
│   ▼        ▼        ▼        ▼        ▼                      │
│ Pessimo  Ruim    Neutro    Bom    Excelente                 │
│                                                              │
│                      META: 4.5+                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Fatores que Afetam o CSAT

| Aumenta CSAT | Diminui CSAT |
|--------------|--------------|
| Resolver rapido | Demorar muito |
| Ser cordial | Ser frio |
| Resolver de vez | Fazer cliente voltar |
| Comunicar bem | Ser confuso |
| Surpreender positivamente | Decepcionar |

### Dicas para CSAT Alto

1. **Sempre agradecer** - Mesmo em reclamacoes
2. **Usar o nome** - Personaliza o atendimento
3. **Ser proativo** - Antecipar necessidades
4. **Mostrar empatia** - Entender o lado do cliente
5. **Resolver de vez** - Evitar que volte

## Lidando com Feedback Negativo

### Quando Receber Critica

1. **Nao leve pro pessoal** - E sobre o servico, nao sobre voce
2. **Escute ativamente** - Deixe o cliente falar
3. **Agradeca o feedback** - E uma oportunidade de melhorar
4. **Tome acao** - Mostre que vai fazer algo a respeito
5. **Acompanhe** - Faca follow-up se possivel

## Checklist de Qualidade

### Para Cada Atendimento

- [ ] Li toda a mensagem do cliente
- [ ] Verifiquei o historico
- [ ] Respondi de forma completa
- [ ] Usei o nome do cliente
- [ ] Confirmei que resolvi a duvida
- [ ] Ofereci ajuda adicional
- [ ] Documentei informacoes importantes
- [ ] Encerrei de forma cordial

### Antes de Transferir

- [ ] Coletei todas as informacoes
- [ ] Criei nota interna completa
- [ ] Avisei o cliente sobre a transferencia
- [ ] Escolhi o departamento correto

### Antes de Resolver Ticket

- [ ] Problema foi realmente resolvido
- [ ] Cliente confirmou satisfacao
- [ ] Documentei a solucao
- [ ] Ofereci canal para retorno

## Fazer vs Nao Fazer

| Fazer | Nao Fazer |
|-------|-----------|
| Resolver na primeira tentativa | Deixar cliente voltar |
| Dar resposta completa | Responder pela metade |
| Documentar tudo | Confiar na memoria |
| Personalizar atendimento | Ser robotico |
| Fazer follow-up | Esquecer do cliente |
| Antecipar necessidades | Responder so o perguntado |
| Agradecer sempre | Ser seco |
| Aprender com feedback | Ignorar criticas |
